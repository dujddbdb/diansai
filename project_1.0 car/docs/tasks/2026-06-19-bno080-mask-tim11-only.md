# BNO08x读包期间仅屏蔽TIM11控制环

**日期**: 2026-06-19

## 涉及文件
- `bsp/bno080.c`
- `bsp/track_config.h`

## 修改前状态
- 曾尝试在 BNO08x I2C 读包期间停止 TIM11/TIM7，避免控制中断打断 BNO08x 时序。
- 实车反馈这种方式会影响速度环，速度闭环不稳定。
- 也讨论过启用 TIM10 最高优先级独立读取，但 TIM10 过高优先级会反复打断其他控制中断，也有实时性风险。

## 修改内容和原因
- 保持当前配置为主循环轮询 BNO08x：`GYRO_TIM10_ENABLE=0`、`GYRO_MAIN_POLL_ENABLE=1`。
- `bno080_update()` 真正读取 SHTP 包前，仅临时关闭 TIM11 更新中断，读完或失败后恢复。
- 不再关闭 TIM7，速度环持续运行，避免速度闭环被 BNO08x 读包破坏。
- 不启用 TIM10 最高优先级读取，避免高频抢占 TIM11/TIM7。

## 测试建议
- 下载后观察直角 `ph1` 时 `yaw` 是否连续更新、`st` 是否不再持续增加。
- 同时观察车速是否比“停止 TIM7”版本稳定。
- 如果 BNO08x 仍卡死，下一步应继续缩短 I2C 单次读包耗时，或将 BNO08x 改为更轻量的分段非阻塞读取，而不是提高 TIM10 抢占频率。
