# 关闭TIM10运行期读取隔离疯转

**日期**: 2026-06-19

## 涉及文件
- `bsp/track_config.h`
- `app/main.c`
- `bsp/track.c`

## 修改前状态
- 用户反馈：不管灰度读到什么数据，只要当前陀螺仪方案运行，小车仍然疯转。
- 当前已确认 `GYRO_CONTROL_ENABLE=0`，陀螺仪不参与控车；`RIGHT_ANGLE_ENABLE=0`，直角状态机不参与控车。
- 因此下一步优先隔离 BNO080 的 TIM10/I2C 运行期读取是否抢占控制周期或干扰速度环。

## 修改内容和原因
- 新增 `GYRO_TIM10_ENABLE=0`，用于临时禁止启动 TIM10 运行期读取 BNO080。
- `app/main.c` 中 BNO080 初始化成功后不再启动 `BNO080_TIM10_Init()`，并打印 `[MAIN] TIM10 gyro runtime disabled`。
- 等待首帧陀螺仪数据的循环增加 `GYRO_TIM10_ENABLE` 条件，避免 TIM10 关闭时卡住 3 秒。
- 保留上一版灰度安全保护：全白/全黑异常帧时误差归零，防止历史误差继续打满方向。
- 串口输出降为 200ms 一行核心数据，并关闭 ADC 细节大行，减少 115200 串口阻塞对主循环采样的影响。

## 测试建议
- 下载后看启动串口是否出现 `[MAIN] TIM10 gyro runtime disabled`。
- 如果这版不疯转，说明疯转来源高度指向 TIM10/BNO080 I2C 运行期读取抢占或阻塞。
- 如果这版仍疯转，则说明不是 TIM10，需要继续查速度环目标/PWM方向/编码器反馈链路。
- 当前版本 `yaw/rt` 不会运行期刷新，这是预期现象；此版是隔离测试固件，不是最终比赛固件。
