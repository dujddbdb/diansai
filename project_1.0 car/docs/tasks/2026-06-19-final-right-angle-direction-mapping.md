# 最终确认直角方向映射

**日期**: 2026-06-19

## 涉及文件
- `bsp/track.c`
- `bsp/grayscale.h`
- `bsp/track_config.h`

## 修改前状态
- 用户确认：刚开始传感器软件和硬件映射都没问题，不需要改灰度映射。
- 用户确认：左转时航向角增大，因此 `GYRO_YAW_DIRECTION` 应保持 `1`。
- 串口位模式 `00001111` 表示左边有黑线。
- 当前问题应收敛为：直角入口位模式到 `is_right_angle` 类型的映射错误。

## 修改内容和原因
- 保持 `GRAY_DIRECTION=1`，不再修改灰度通道映射。
- 保持 `GYRO_YAW_DIRECTION=1`，符合左转 yaw 增大的实车现象。
- 仅交换 `Track_Check_Right_Angle()` 中两个入口模式的 `current_type` 赋值：
  - `ir_raw[5..7]` 黑线模式赋 `current_type=1`
  - `ir_raw[0..2]` 黑线模式赋 `current_type=2`
- 执行逻辑保持原样：`type=1` 对应右转目标，`type=2` 对应左转目标。

## 测试建议
- 串口出现 `00001111` 时，应锁存为左转对应的 `type=2`。
- 左直角执行时 yaw 应向增大方向变化。
- 如果锁存类型正确但实车方向仍反，再检查电机左右输出，而不是灰度映射。
