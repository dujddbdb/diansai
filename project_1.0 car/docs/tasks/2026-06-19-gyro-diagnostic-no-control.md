# 陀螺仪诊断但不参与控车

**日期**: 2026-06-19

## 涉及文件
- `bsp/track_config.h`
- `bsp/track.c`

## 修改前状态
- 纯灰度循迹测试稳定，说明灰度、编码器、电机速度环基本正常。
- 上电疯转只在陀螺仪参与后出现，下一步需要观察 BNO080 的 `yaw/rt` 数据，但不能让它继续影响控车。

## 修改内容和原因
- 将 `GYRO_ENABLE` 重新设为 `1`，让 BNO080 初始化、TIM10排空、TIM11读取和串口诊断恢复。
- 新增 `GYRO_CONTROL_ENABLE=0`，用于关闭陀螺仪控制输出。
- `Track_Check_Right_Angle()` 中记录 yaw 目标、yaw 角退出检测、直角 yaw PID、直线 gyro correction 都改为受 `GYRO_ENABLE && GYRO_CONTROL_ENABLE` 控制。
- 当前固件效果：能看陀螺仪串口数据，但小车控制仍走灰度/灰度直角回退逻辑。

## 测试建议
- 下载后观察车是否仍稳定；若稳定，说明疯转来自陀螺仪控制输出而不是初始化本身。
- 串口重点看静止时 `yaw` 是否缓慢漂移、`rt` 是否有大跳变。
- 手动左转车体时，确认 `yaw` 是否增大；若不增大，再考虑 `GYRO_YAW_DIRECTION`。
- 若 `rt` 静止时大幅跳动，需要优先处理 yaw 差分/数据帧率/滤波，而不是加大 PID。
