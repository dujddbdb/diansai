# 降低直线陀螺仪补偿抑制上电摇摆

**日期**: 2026-06-19

## 涉及文件
- `bsp/track_config.h`
- `bsp/track.c`

## 修改前状态
- 上电后小车出现持续左右摇摆。
- 直线循迹中陀螺仪补偿使用 `gyro_correction = -KD_GYRO_STRAIGHT * gyro_yaw_rate`。
- 当前 `KD_GYRO_STRAIGHT` 为 `1.0f`，虽然不算极大，但 BNO080 上电初期 yaw 差分噪声、车体轻微抖动、速度环响应叠加后可能放大成左右摆动。
- 直线陀螺仪补偿没有单独限幅，噪声尖峰会直接进入左右目标转速差。

## 修改内容和原因
- 将 `KD_GYRO_STRAIGHT` 从 `1.0f` 降到 `0.2f`，先让直线陀螺仪弱参与，便于判断摇摆是否来自陀螺仪补偿。
- 新增 `GYRO_STRAIGHT_LIMIT = 5.0f`，限制直线陀螺仪最多贡献 ±5 RPM 差速。
- 在正常循迹分支对 `gyro_correction` 做 `f_clamp()` 限幅，避免 yaw 噪声尖峰造成左右突然摆动。

## 测试建议
- 上电后先架空车轮，看串口 `rt` 是否在静止时接近 0；若 `rt` 明显跳动，说明 BNO080 yaw 差分噪声较大。
- 下载后实车直线测试：若摇摆明显减轻，说明原直线陀螺仪补偿过强或噪声被放大。
- 若仍左右摇摆，下一步优先检查灰度 `err/PID` 是否来回跳，或临时将 `GYRO_ENABLE` 设为 `0` 对比纯灰度。
- 若变得太迟钝，再逐步把 `KD_GYRO_STRAIGHT` 从 `0.2` 调到 `0.4/0.6`。
