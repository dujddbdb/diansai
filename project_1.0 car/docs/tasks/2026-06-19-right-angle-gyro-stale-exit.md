# 直角执行期陀螺仪停滞退出

**日期**: 2026-06-19

## 涉及文件
- `bsp/track.c`
- `bsp/track_config.h`

## 修改前状态
- 实车串口显示直角执行期保持 `ph1 t2`。
- `yaw:+107.5` 与 `rt:-0.58` 长时间不变化。
- `st` 从约 `1590` 持续增长到 `2310`，说明 BNO08x 没有新的旋转向量数据被上层消费。
- 原逻辑只要 `gyro_yaw_available=1` 就优先等待 yaw 达到 `CORNER_YAW_TARGET`，因此 yaw 停住后会卡在直角执行状态。

## 修改内容和原因
- 新增 `RIGHT_ANGLE_GYRO_STALE_EXIT_MS=300U`。
- 进入 `ph1` 时清零 `gyro_stale_ms`，避免携带旧停滞时间。
- 直角执行期若 `gyro_stale_ms` 超过阈值，直接进入退出流程，不再等待 yaw 达标。
- 直角执行期若灰度重新检测到黑线，也允许退出，避免全白后 yaw 停滞导致永久不退出。

## 当前判断
- 日志证明直角时 BNO08x 数据流在上层停止刷新。
- 可能原因包括 INTN/I2C 读包链路停滞、电机转弯时电源/干扰影响 BNO08x、或读取到了非旋转向量包但没有产生 `new_data`。
- 当前补丁先保证车不会因为 BNO08x 停滞卡死在 `ph1`。

## 编译验证
- Keil MDK 构建通过：`0 Error(s), 0 Warning(s)`。

## 测试建议
- 再测直角时观察 `st`：如果进入 `ph1` 后 `st` 增到约 `300`，应自动退出到 `ph3/ph0`。
- 如果仍想定位根因，下一步串口增加 `g_exti5_count` 与 `gyro_frame_count`，区分是 INTN 没来，还是包来了但没解析成 yaw。
