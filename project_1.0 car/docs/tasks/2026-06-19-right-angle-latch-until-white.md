# 直角方向锁存并等待全白执行

**日期**: 2026-06-19

## 涉及文件
- `bsp/track.c`
- `bsp/track.h`
- `bsp/track_config.h`
- `app/main.c`

## 修改前状态
- 上一版把当前帧直角类型改为局部变量后，避免了旧状态无限残留。
- 但真实赛道中“直角入口特征”和“8路全白确认”不一定连续多帧同时稳定出现。
- 如果入口只检测到一次，下一帧就变为全白，局部 `current_type` 会重新变为 0；若状态机没有提前锁存方向，就可能无法按正确方向执行直角。
- 之前加入的预触发超时不符合当前调车策略：用户希望不加超时，只要检测过直角方向，就等全白再执行。

## 修改内容和原因
- Phase 0 只要检测到左/右直角入口，就立即锁存 `is_right_angle` 和 `right_angle_detect_type`，并进入 Phase 2。
- Phase 2 不再因为当前帧没有直角入口特征而回退；它持续等待 8 路全白。
- Phase 2 若再次看到直角入口，会刷新锁存方向，避免左右方向误判。
- 删除 `right_angle_pretrigger_timer` 和 `RIGHT_ANGLE_PRETRIGGER_TIMEOUT`，不再使用超时回退。

## 测试建议
- 串口观察：入口特征出现后应进入 `ph=2`，即使后续入口特征消失，也应保持等待全白。
- 全白连续达到 `RIGHT_ANGLE_CONFIRM_CNT` 后应进入 `ph=1` 执行直角。
- 如果误触发变多，再考虑收紧入口特征，不建议恢复超时。
