# 2026-06-20 - 降低 BNO08x 读取频率缓解速度环卡顿

## 日期
- 2026-06-20

## 涉及文件
- `bsp/track_config.h`

## 修改前状态
- BNO08x 读取周期为 10ms/100Hz。
- I2C 读取期间会临时屏蔽 TIM7/TIM11 更新中断并保留 pending。
- 实车感觉速度环运行有点慢、车一卡一卡，推测 BNO08x I2C 读包对 5ms TIM7 速度环造成周期抖动。

## 修改内容和原因
- 将 `GYRO_MAIN_POLL_PERIOD_MS` 从 `10U` 改为 `20U`。
- BNO08x 读取频率从 100Hz 降到 50Hz，减少 I2C 事务打断 5ms 速度环的次数。
- 未修改速度环 PI 参数，先优先降低调度干扰，避免把时序问题误调成 PID 参数问题。

## 当前时序
- TIM7 速度环：5ms/200Hz 不变。
- TIM11 控制环：1ms 不变。
- BNO08x 读取请求：由 TIM11 计数到 20ms 后置位，主循环执行 I2C 读取。

## 测试建议
- 下载后观察“卡一卡”的感觉是否明显减轻。
- 如果卡顿减轻但直角角度反馈变慢，可试 `GYRO_MAIN_POLL_PERIOD_MS = 15U` 折中。
- 如果仍卡顿，下一步不要先动速度 PI，建议限制单次 `bno080_update()` 只读一个短包或改 I2C 非阻塞/DMA。
